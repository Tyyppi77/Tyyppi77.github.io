GunHero

Previous Next

“Simple in its mechanics but beautifully fiendish in the way it puts them all together, it’s an almost perfect blend of challenge vs. accomplishment.” 4/5 - Indie Game Reviewer

GunHero is a 2D action-platformer-shooter that I developed using a custom C++ framework built ontop of SDL2. The development process of the game took about 2.5 years from first prototype to initial release on Steam. The game has been updated post release to for an example include support for custom player created levels shared using the Steam Workshop.

The game consists of six differently themed areas with various obstacles and mechanics in each of them. Each area contains also a boss fight as the last level. There are a little over 125 levels in the game, built using the in-game level editor that was later released for public use.

The codebase for the game is currently around 100 000 lines. SDL2 handles the window management, input handling, rendering and audio on a low level, but the rest is handled by the custom code written ontop of that.

ImGuiSDL

ImGuiSDL is a renderer for Dear ImGui that allows Dear ImGui to be rendered using a SDL_Renderer.

By default, Dear ImGui is designed to be easily rendered using any modern 3D rendering API, like OpenGL or DirectX, by sending vertex and index buffers to a simple shader. While the modern SDL2 rendering pipeline does use hardware acceleration behind the scenes, it does not expose a cross-platform interface to passing in generic vertex data. That's why I built ImGuiSDL.

ImGuiSDL performs the rendering in two steps. First, it tries to find 6 adjacent vertices that form a rectangle that can be rendered in a quick fashion even if it is textured. If performing a rectangle rendering isn't possible, ImGuiSDL falls back to a fixed-point triangle rasterizer (that's not perfect but good enough for debug use, just like Dear ImGui is intended to be used). Because the triangle rasterization is slow, the result is cached into a render texture so that following frames can quickly just do a texture draw to the screen.

ImGuiSDL is implemented in C++ in a easy-to-use manner: to use it, one needs to add two files to their build, and call one function to initialize the library and another to render.
